WikiSearch Project

I worked on the development of WikiSearch, a search engine that integrates SQL and NoSQL databases to process and query Wikipedia's XML dumps. The project involved full-stack development, data processing, and API integration. On the back-end, we built a Flask API in Python, deployed using Docker, and connected to multiple databases:

  • Mongo Atlas (NoSQL) with optimized search indexes.
  • Oracle Autonomous Database (SQL) for structured data storage.
  • Oracle NoSQL Database Cloud for logging user queries.
  • Firebase Real-Time Database for authentication.

The system included a data loader, developed in Python, which processed Wikipedia's XML dumps stored in Object Storage and efficiently loaded structured data into the databases. The frontend was implemented in React. Additionally, the project was deployed and automated in Oracle Cloud Infrastructure (OCI) using Terraform.

MovieSearch with Neo4j

I worked on the development of MovieSearch, a platform for movie search and recommendations utilizing document-oriented and graph databases. The project involved full-stack development, database integration, and cloud deployment. On the backend, we built a Python-based API, and integrated with MongoDB Atlas (NoSQL) for movie storage and Neo4J (graph database) for relationship-based recommendations. The mobile application was developed using Thunkable X, with user authentication managed via Firebase. Connectivity was enhanced using ngrok for seamless API exposure.
